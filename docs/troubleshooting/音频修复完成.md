## 问题总结

### 原始问题

- ❌ 数字人 TTS 播放没有声音
- ❌ 数字人嘴巴不动（没有口型驱动）

### 根本原因

音频流向混乱，导致音频要么只发送到 WebRTC（有声音无口型），要么只发送到 LipASR（有口型无声音），无法同时工作。

---

## 修复内容

### 修改的文件

1. ✅ **ttsreal.py** (DoubaoTTS 类)

   - 移除 `direct_to_webrtc` 判断逻辑
   - 统一通过 `parent.put_audio_frame()` 处理音频
   - 简化音频转发流程

2. ✅ **basereal.py** (BaseReal 类)
   - 优化 `put_audio_frame()` 方法
   - 确保音频同时转发到 LipASR 和 WebRTC
   - 改进音频块大小处理（填充而不是丢弃）
   - 提高队列容量阈值（60→100）

### 核心改进

```
修复前：
DoubaoTTS → 判断direct_to_webrtc
            ├─ True → 直接发WebRTC (有声音，无口型)
            └─ False → 发parent (可能有口型，可能无声音)

修复后：
DoubaoTTS → parent.put_audio_frame()
            ├─→ LipASR (口型驱动) ✅
            └─→ WebRTC (声音播放) ✅
```

## 技术细节

### 音频处理流程

1. **DoubaoTTS** 通过 WebSocket 接收 PCM 音频（16kHz, int16）
2. 转换为 float32 格式（归一化到-1.0~1.0）
3. 分块为 320 样本/块（20ms @ 16kHz）
4. 调用 `parent.put_audio_frame(chunk, eventpoint)`
5. **BaseReal** 接收后：
   - 转发到 **LipASR** → 提取音频特征 → 驱动口型
   - 转发到 **WebRTC** → 转换为 AudioFrame → 浏览器播放

### 关键代码位置

- `ttsreal.py:750-790` - DoubaoTTS 音频推送
- `basereal.py:155-250` - BaseReal 音频处理
- `lipasr.py:32-60` - LipASR 特征提取

## 故障排查

### 如果还是没有声音

1. **检查环境变量**

   ```bash
   echo $DOUBAO_APPID
   echo $DOUBAO_TOKEN
   echo $DOUBAO_VOICE_ID
   ```

2. **检查 WebRTC 连接**

   - 浏览器是否授权音频权限
   - 网络连接是否正常
   - 查看浏览器控制台是否有错误

3. **查看详细日志**
   ```bash
   export LOG_LEVEL=DEBUG
   python app.py
   ```

### 如果口型还是不动

1. **检查 LipASR 初始化**

   ```bash
   grep "LipASR" livetalking.log
   ```

2. **检查音频转发**

   ```bash
   grep "Audio forwarded to LipASR" livetalking.log
   ```

3. **检查模型文件**
   - 确认 `models/wav2lip.pth` 存在
   - 确认模型加载成功
