# DoubaoTTS 音频播放和口型驱动修复说明

## 问题现象

- ❌ 数字人 TTS 播放没有声音
- ❌ 数字人嘴巴不动（没有口型驱动）

## 修复内容

已修复以下关键文件：

1. **ttsreal.py** - DoubaoTTS 音频生成和转发逻辑
2. **basereal.py** - 音频处理和双路转发（LipASR + WebRTC）

## 核心改进

### 1. 统一音频处理流程

```
DoubaoTTS → parent.put_audio_frame() → BaseReal
                                          ├─→ LipASR (口型驱动)
                                          └─→ WebRTC (声音播放)
```

### 2. 关键修复点

- ✅ 移除了混乱的 `direct_to_webrtc` 判断逻辑
- ✅ 确保音频同时转发到 LipASR 和 WebRTC
- ✅ 改进音频块大小不匹配的处理（填充而不是丢弃）
- ✅ 优化队列管理，减少音频丢失
- ✅ 增强日志输出，便于调试

## 技术细节

### 音频处理流程

1. **DoubaoTTS 生成音频**

   - 通过 WebSocket 接收 PCM 音频数据
   - 采样率: 16kHz
   - 格式: int16 → float32 (归一化到 -1.0 ~ 1.0)

2. **分块处理**

   - 每块 320 个样本 (20ms @ 16kHz)
   - 不足 320 样本时填充静音

3. **双路转发**
   - **LipASR**: 用于提取音频特征，驱动口型
   - **WebRTC**: 转换为 AudioFrame，发送到浏览器播放
