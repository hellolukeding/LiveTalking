## 启动方式

### 方式一：使用启动器（推荐）

```bash
# 1. 安装依赖（如果尚未安装）
poetry install

# 2. 激活虚拟环境
poetry shell

# 3. 运行启动器
python run_livetalking.py

# 4. 选择 "1. 快速启动 (推荐)"
```

### 方式二：直接运行修复版脚本

```bash
# 1. 激活虚拟环境
poetry shell

# 2. 设置Python路径并运行
export PYTHONPATH="${PYTHONPATH}:$(pwd):$(pwd)/src:$(pwd)/src/core:$(pwd)/src/llm:$(pwd)/src/utils:$(pwd)/src/main"
python src/main/start_quick_fixed.py
```

### 方式三：使用 Poetry 运行（推荐）

```bash
# 1. 安装依赖
poetry install

# 2. 激活虚拟环境
poetry shell

# 3. 直接运行
poetry run livetalking

# 或者在虚拟环境中直接运行
livetalking
```

## 环境变量配置

在 `.env` 文件中配置以下参数：

```bash
# TTS服务配置
TTS_TYPE=edge                    # 可选: edge, doubao, tencent, azure
EDGE_TTS_VOICE=zh-CN-YunxiNeural # Edge TTS语音
DOUBAO_VOICE_ID=zh_female_xiaohe_uranus_bigtts  # 豆包TTS语音ID
TENCENT_VOICE_TYPE=1001          # 腾讯TTS语音类型
AZURE_VOICE_NAME=zh-CN-XiaoxiaoNeural  # Azure TTS语音

# ASR服务配置
ASR_TYPE=lip                     # 可选: lip, tencent, funasr, huber

# Web服务配置
LISTEN_PORT=8011                 # Web服务端口
MAX_SESSION=1                    # 最大会话数

# LLM配置（如果使用聊天功能）
OPENAI_API_KEY=your_api_key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-3.5-turbo
```

## 模型文件准备

确保以下模型文件存在于 `models/` 目录：

- `wav2lip256.pth` - Wav2Lip 模型
- `shape_predictor_68_face_landmarks.dat` - 人脸关键点检测
- `s3fd.pth` - 人脸检测模型

## 访问前端

启动成功后，访问以下地址：

- **集成前端**: http://localhost:8011/dashboard.html
- **WebRTC API**: http://localhost:8011/webrtcapi.html
- **其他页面**: http://localhost:8011/

## 常见问题

### 1. 导入错误

**问题**: `ModuleNotFoundError: No module named 'logger'`

**解决**: 使用启动器 `python run_livetalking.py` 会自动设置 Python 路径

### 2. 模型文件缺失

**问题**: `模型加载失败: 文件不存在`

**解决**: 下载模型文件到 `models/` 目录

### 3. 端口被占用

**问题**: `Address already in use`

**解决**: 修改 `.env` 中的 `LISTEN_PORT=8012` 或其他端口

### 4. 依赖未安装

**问题**: `ImportError: No module named '...'`

**解决**:

```bash
poetry install
poetry shell
```

## 服务启动顺序

1. **TTS 服务** (如果使用外部服务)

   - 豆包 TTS: 需要启动豆包服务
   - 腾讯 TTS: 需要配置 API 密钥
   - Edge TTS: 免费，无需额外配置

2. **ASR 服务** (如果使用外部服务)

   - 腾讯 ASR: 需要配置 API 密钥
   - Lip ASR: 本地服务，无需配置

3. **LiveTalking 主服务**
   - 运行 `run_livetalking.py`
   - 选择快速启动模式
