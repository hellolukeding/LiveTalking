## 🚀 启动方式（3 种选择）

### 方式一：Poetry 命令（最推荐）

```bash
# 1. 安装依赖
poetry install

# 2. 启动项目
poetry run livetalking
```

**注意**: 无需激活虚拟环境，poetry 会自动处理。首次运行会提示配置，按提示操作即可。

### 方式二：启动器脚本

```bash
python run_livetalking.py
# 然后选择 "1. 快速启动 (推荐)"
```

### 方式三：直接运行修复版

```bash
export PYTHONPATH="${PYTHONPATH}:$(pwd):$(pwd)/src:$(pwd)/src/core:$(pwd)/src/llm:$(pwd)/src/utils:$(pwd)/src/main"
python src/main/start_quick_fixed.py
```

## ⚙️ 环境配置

在 `.env` 文件中配置：

```bash
# TTS服务
TTS_TYPE=edge                    # edge/doubao/tencent/azure
EDGE_TTS_VOICE=zh-CN-YunxiNeural

# ASR服务
ASR_TYPE=lip                     # lip/tencent/funasr/huber

# Web服务
LISTEN_PORT=8011
MAX_SESSION=1

# LLM (可选)
OPENAI_API_KEY=your_key
OPENAI_BASE_URL=https://api.openai.com/v1
```

## 📝 注意事项

1. **首次使用**:

   - 确保已安装 Poetry
   - 运行 `poetry install` 安装依赖
   - 配置 `.env` 文件

2. **模型文件**:

   - 需要手动下载模型文件到 `models/` 目录
   - 可以从原项目或相关链接下载

3. **服务依赖**:

   - Edge TTS: 免费，无需配置
   - Lip ASR: 本地服务，无需配置
   - 其他服务: 需要 API 密钥

4. **端口冲突**:
   - 默认端口: 8011
   - 可在 `.env` 中修改 `LISTEN_PORT`

## 🔧 故障排除

### 问题: `ModuleNotFoundError`

**解决**: 使用 `poetry run livetalking` 或 `python run_livetalking.py`

### 问题: `模型加载失败`

**解决**: 下载模型文件到 `models/` 目录

### 问题: `端口被占用`

**解决**: 修改 `.env` 中的 `LISTEN_PORT`

### 问题: `导入错误`

**解决**:

```bash
poetry install
poetry shell
```
